# GigaNews - новостное приложение

## Приложение создано через Android Studio на языке Kotlin (Jetpack Compose) с использованием публичного новостного API ["NewsAPI"](https://newsapi.org/). Cоответствует архитектурной модели MVVM.

## Основные функции:
### 1. Список новостей (LazyColumn)
- каждая новость - отдельная кликабельная карточка с фото, заголовком и кратким описанием
- описание ограничено 3-мя строками
- если изображение не загружается или отсутствует, оно заменяется на заглушку с названием приложения
### 2. Детали статьи
- страница статьи содержит изображение, заголовок, краткое и подробное описание
- есть кнопка для возврата к ленте
- идентично карточке в списке новостей, картинка может заменяться на заглушку
### 3. Обработка ошибок/валидация и т.п.
- при появлении ошибки на этапе загрузки списка новостей отображается ее код и кнопка "Повторить загрузку"
- при прокрутке списка карточки с новостями и их изображения постепенно подгружаются, уменьшая нагрузку
- функции приложения распределенны по назначению на разные файлы .kt , что обеспечивает стабильность его работы

## Особенности приложения:
- Полностью функционирующее приложение
- Собственные логотип, название и изображение-заглушка
- (для разработчиков) Удобная структура для последующей модификации


## Важные уточнения:
1. Из-за особенности NewsAPI новости из всех доступных в API регионов, кроме США (us), отсутствуют (если попытаться получить новости из РФ (ru), Колумбии (co) или любого другого региона, то ошибки не возникнет, но будет получен пустой словарь, из-за чего пришлось использовать новости США для демонстрации работы приложения).
2. Функция поиска новостей не была реализована из-за возникших трудностей с изучением Jetpack Compose и самим API (реализация этой функция возможна, если верить офицальной документации NewsAPI, и в перспективе она не является слишком сложной, просто мне не хватило навыков и времени).
3. ~~Опять~~ Из-за особенностей NewsAPI список новостей не является бесконечным - он сам ограничивает количество получаемых новостей. Примерно по той же причине подробное описание не совсем подробное - API отправляет строку-описание, которая прерывается надписью типа [2447 символов].
